# Threat Hunting Queries – Wazuh SIEM

## Overview

This document contains the queries used during threat hunting to identify suspicious behavior generated by Atomic Red Team simulations.

---

## Methodology

Queries were created based on observed behavior from executed MITRE ATT&CK techniques:

* Process execution
* Authentication activity
* System reconnaissance
* Suspicious command usage

---

## 1.- Process Creation (Event ID 4688 OR 600)

### Purpose

Identify newly created processes that may indicate malicious activity.

### Query

data.win.system.eventID:4688 OR data.win.system.eventID:600

---

## 2.- Suspicious Processes

### Purpose

Detect potentially malicious or suspicious commands.

### Query

powershell OR cmd.exe OR tasklist

---

## 3.- Authentication Activity

### Purpose

Monitor login attempts and potential brute-force behavior.

### Query

data.win.system.eventID:4624 OR data.win.system.eventID:4625

---

## 4.- Failed Login Attempts (Brute Force Indicator)

### Purpose

Identify repeated failed login attempts.

### Query

data.win.system.eventID:4625

---

## 5.- Successful RDP Login

### Purpose

Detect remote access to the system.

### Query

data.win.system.eventID:4624 AND data.win.eventdata.logonType:10

---

## 6.- Activity by Specific User

### Purpose

Track behavior of a targeted account.

### Query

data.win.eventdata.targetUserName:testuser

---

## 7.- Activity from Specific IP

### Purpose

Analyze activity originating from a single IP address.

### Query

data.win.eventdata.ipAddress:192.168.56.X

---

## 8.- PowerShell Activity (T1059 / T1003 Related)

### Purpose

Detect suspicious PowerShell usage.

### Query

powershell

---

## 9.- System Discovery Activity (T1082)

### Purpose

Identify system information gathering behavior.

### Query

systeminfo OR hostname

---

## 10.- Process Enumeration (T1057)

### Purpose

Detect enumeration of running processes.

### Query

tasklist

---

## Notes

* Queries should be combined with time filters for better analysis
* Always correlate multiple events for accurate conclusions
* Context is critical in threat hunting
